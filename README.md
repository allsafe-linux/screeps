### Screeps World 玩家脚本 - 多房间自动化运营框架 - 开源介绍
这个 Screeps World 玩家脚本框架，是我跟豆包啰里八嗦了一个月生成的。然后专为多房间规模化运营设计，兼顾新手友好性与生产级稳定性，核心聚焦**自动化、可扩展、低维护**三大特性，完美适配从单房间起步到跨房间扩张的全阶段需求。

#### 核心特点与优势

##### 1. 🔄 多房间去中心化架构，独立运营不耦合
- **核心设计**：每个自有房间独立执行经济、防御、建造逻辑，主房间仅作为远程资源调度和扩张控制中心，避免单房间故障导致全局瘫痪
- **角色隔离**：所有基础角色（矿工/搬运工/防御者等）通过「角色名+房间名」后缀区分（如 `miner_W13S59`），不同房间的 creep 各司其职，无资源争抢问题
- **动态适配**：新占领房间自动继承完整经济逻辑，无需手动修改配置即可独立孵化 creep、维护能源循环

##### 2. ⚡ 极致优化的能源管理体系
- **矿工负载均衡**：自动计算各能量源的矿工数量，动态分配最优采集目标，避免单源扎堆、多源闲置的效率损耗
- **Link 智能调度**：支持 Source/Spawn/Controller 三链路协同，保底保障控制器升级能量，剩余能量按缺口比例动态分配，兼顾生产与升级
- **能量优先级管理**：搬运工严格遵循「Spawn/Extension → Tower → Storage → Terminal」送能逻辑，应急模式下优先保障基础生产

##### 3. 🚀 自动化扩张与远程资源开发
- **智能殖民逻辑**：自动占领未控制房间 → 升级控制器 → 选址建造 Spawn → 部署基础经济体系，全流程无需人工干预
- **远程房间风控**：远程开采前自动计算收益（能源产出 - creep 成本），收益为负时自动暂停；检测到敌对单位立即关停远程作业
- **扩张 Worker 专精**：新房间优先建造 Spawn，按「Extension → Road → Container → Tower → Storage」顺序自动补全基础设施

##### 4. 🛡️ 高可靠性的应急与防御机制
- **应急模式触发**：当房间即时能量 < 300 且 Storage 能量 < 1000 时，自动切换应急模式，优先孵化低成本 creep，停止非必要建造
- **Creep 生命周期管理**：低寿命 creep 自动收尾能量，应急模式下禁止自杀，减少资源浪费；远程保留者自动续期，防止房间控制权丢失
- **双塔协同防御**：Tower 优先攻击入侵者 → 治疗受伤 creep → 修复关键建筑 → 维护墙/壁垒，保障房间安全

##### 5. 💎 Pixel 自动化与经济贸易系统
- **安全生成 Pixel**：自动检查冷却和 CPU 上限，购买 Unlimited CPU DLC 后可每 100 Tick 稳定生成 1 个 Pixel
- **模拟抽卡功能**：支持自定义抽卡次数，按概率分配不同稀有度奖励，可直接对接官方商店 API
- **智能市场贸易**：Terminal 自动出售过剩 Z 矿（保底 2500 库存）、补充不足 H 矿（上限 5000），只执行高于/低于市场价的交易，保证收益

##### 6. 🛠️ 易扩展、易维护的工程化设计
- **模块化拆分**：按功能拆分为「内存清理/角色逻辑/设施管理/经济系统」等独立模块，代码结构清晰，便于局部修改
- **配置中心化**：所有阈值（应急能量、墙维修上限、贸易价格等）集中在顶部配置区，无需深入代码即可调整策略
- **完善的错误处理**：关键操作均有结果校验和日志输出，API 调用错误（如 CPU 不足、冷却中）友好提示，便于问题定位

#### 适用场景
- 新手玩家：开箱即用，无需复杂配置即可实现单房间自动化运营
- 进阶玩家：支持多房间扩张和远程开采，可快速搭建跨房间资源网络
- 付费玩家：购买 Unlimited CPU DLC 后，Pixel 生成模块可稳定产出，配合抽卡系统实现资源兑换

#### 核心优势总结
1. **去中心化多房间架构**：各房间独立运营，扩展性强，故障影响范围可控
2. **智能资源调度**：从采集、传输到分配全链路优化，能源利用效率最大化
3. **全自动化运营**：从单房间维护到跨房间扩张，99% 操作无需人工干预
4. **高鲁棒性**：完善的应急机制和错误处理，极端情况下仍能保障基础生产
